# Product Guidelines: Brewery Explorer

## 1. Visual Style
The visual style of the application should be "Fun & Curious". This will be implemented using a light and engaging theme from daisyUI, such as "cupcake" or "retro". The interface must be clean, intuitive, and visually appealing.

## 2. Tone of Voice
- **Friendly and Accessible:** All text within the application, including labels, messages, and descriptions, should be written in a friendly and approachable tone.
- **Language:** All user-facing text should be clear and easy to understand.

## 3. User Experience (UX)
- **Consistency:** The user experience must be consistent across all features. Navigation, component behavior, and visual elements should follow predictable patterns.
- **Intuitive Design:** The application should be easy to use without requiring a manual. Users should be able to discover features and complete tasks intuitively.

## 4. Accessibility
The project will prioritize accessibility to ensure the application is usable by everyone. This includes considering aspects like color contrast, keyboard navigation, and screen reader compatibility.

## 5. Code and Documentation Standards
- **Language:** All internal documentation, including code comments and commit messages, MUST be written in **SPANISH**. This is a strict requirement for the academic evaluation of the project.
